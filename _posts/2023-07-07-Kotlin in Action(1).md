코틀린 인 액션 책으로 스터디를 하게 됐는데, 내가 이해한 것을 기반으로 잘 기록해둘 계획이다.

## 개요

### 코틀린은 정적타입 지정언어다

정적 타입(statically typed) 지정이라는 말은 모든 구성요소의 타입을 컴파일 때 알 수 있고, 객체의 필드/메서드를 사용 시 컴파일러가 타입을 검증한다는 뜻이다. 동적 타입은 타입과 무관하게 모든 값을 변수에 넣을 수 있고, 검증시점이 실행시점이다. Groovy가 대표적인데, 간결한 코드는 확실한 장점이지만 컴파일 시 걸러내지 못하는 오류가 실행시점에 발생하기 때문에 각각 장단점이 있다. 

정적타입 지정언어의 장점은 성능, 신뢰성(컴파일러 검증), 유지보수성, 도구 지원 등이 있다.

코틀린은 자바와 같은 정적 타입 지정 언어지만, 컴파일러가 문맥으로 타입을 추론한다. 따라서 타입 명시가 따로 필요하진 않지만 유지 보수시 가독성을 위해 명시를 해 주는게 더 좋은 방향이다.

마지막으로 코틀린은 nullable type을 지원하기 때문에 컴파일 시 null 검사를 거칠 수 있다. 또 function type을 지원하기 때문에 함수형 프로그래밍을 지원한다. 안드로이드 앱 개발에서도 이 장점은 유효한데, null을 추적하기 때문에 `NPE(NullPointerException)`의 빈도를 확 줄일 수 있게 된다.

### 함수형 프로그래밍

처음 접했을 때 굉장히 난해 했는데, 크게 3가지 특징이 있다. 

- 함수는 **일급시민**(일반 값처럼 사용가능하다. 심지어 반환값으로 함수를 쓸 수 있다.)

- 불변성

- side effect 없음(순수 함수를 사용)

이런 특징은 간결성을 살려줘 중복을 줄여주고, 동기화 걱정이 없어서 다중 스레드에서 사용시 안전하다는 점이다. 또 순수함수를 사용한다는 점은 테스트하기 편하다는 장점도 있다. 

### 코틀린 철학

실용성, 간결성, 안전성, 상호운용성이 있는데, 각 항목을 간단히 정리해봤다.

**실용성**:

이미 다른 언어가 검증한 해법과 기능들을 답습하기 때문에 코틀린을 배우는데 수고가 줄고, IDE에 좋은 도구가 많기 때문에(컴파일된 자바 바이트코드 읽기와 같은) 실용적이라고 할 수 있다.

**간결성**:

보일러 플레이트를 자바에 비해 많이 줄였다. getter, setter는 물론이고 data class와 같은 특수 클래스, 다양한 라이브러리를 이용해 코드의 간결함을 유지한다.

**안전성**:

우선 어느정도 안전성이 JVM을 따르기때문에 입증되어있고, 여기서 NPE에 대한 안전성이 추가된 것이다. 

**상호** **운용성**:

자바의 기존 라이브러리를 그대로 가져다 사용할 수 있고, 자바와 코틀린이 섞인 프로젝트를 컴파일하는데 아무런 장애가 되지않는다. 

### 컴파일 과정

`.kt`로 저장된 파일을 컴파일러가 컴파일을 마치면 `.class`파일을 생성한다. `.class`파일은 `.jar`를 통해 패키징과정을 거쳐 애플리케이션에서 실행된다. 코틀린 컴파일러를 사용한 파일은 코틀린 런타임 라이브러리에 의존한다. 따라서 코틀린으로 컴파일 된 앱을 배포할 때는 코틀린 런타임 라이브러리도 같이 배포해야되는데, gradle과 같은 도구가 앱 패키지 시점에 코틀린 런타임을 포함시켜준다. 이러면 용량이 커지는 게 아닐까? 다행히 코틀린 런타임 시스템의 용량은 작기 때문에 포함 전과 후의 용량 차이가 크게 나지 않는다. 

여기서 한번쯤은 궁금할 수도 있는데, 코틀린은 클래스를 만들지 않고 `fun main(){ }`으로 바로 코드를 실행할 수 있다. 그 이유는 컴파일러가 자동으로 클래스를 만들고 진입점을 만들어주기 때문이다.

## 기본 요소들

### 함수

```kotlin
fun foo(param: Int): String{
    return if(param>10) "BIG" else "SMALL"
}
```

파라미터는 이름 뒤에 타입을 명시한다.

코틀린의 if는 문(statement)이 아니라 식(expression)이다. 즉 값을 반환 할 수 있고 다른 식의 하위 요소로 연산에 관여할 수 있다는 것이다. 자바의 if는 문이지만 코틀린의 if는 식이다.

```kotlin
fun foo(param:Int) = if(param>10) "A" else "B"
```

함수가 식을 본문으로 가질 수 있는데, 이 경우 반환타입을 생략할 수 있다.

### 변수

`val`, `var`로 나뉘는데, val로 선언한 변수는 immutable하고, var는 mutable하다. val로 선언하는 것은 자바의 final 변수로 이해하면 된다. 변수를 초기화 할 때는 변수 타입이 필요 할 수도, 없을 수도 있다. 선언과 동시에 초기화를 한다면, 컴파일러가 타입추론이 가능하지만, 초기화 시점이 선언시점보다 후라면 컴파일러가 타입추론을 할 수 없다. 그래서 초기화를 나중에 할 경우 타입명시가 꼭 필요하다.

처음 코틀린을 접할 때 들었던 의문인데

```kotlin
val arr = arrayListOf("a","b")
```

arr은 immutable하다. 그러나 arr.add("c")로 조작이 가능한데, 왜 그럴까.

val 참조 자체는 불변이다. 하지만 참조가 가리키고 있는 객체의 내부 까지는 val가 관여하지않기 때문에 arr가 가리키고 있는 객체에 대한 연산이라고 보면 된다.

변수를 문자열 안에서 사용하려면 아래와 같이 표기한다. 이를 문자열 템플릿이라고 한다.

```kotlin
print("Hello, $name")
print("Hello, ${name}")
```

이런식으로 $를 붙여서 사용하면 되는데, 중괄호로 묶어주는 게 좋다. 컴파일러가 변수의 범위를 정확하게 인식하게 하기 위함이다. 멤버 변수를 사용하거나, 한글과 영문변수를 붙여 쓸때 컴파일러가 오류를 내게 되므로 중괄호로 묶어서 사용하는 게 권장된다. 변수 뿐만 아니라 식도 중괄호에 넣어 사용할 수 있다.

### 프로퍼티

자바 클래스에서의 필드와 게터, 세터를 한번에 묶어서 코틀린 프로퍼티가 탄생했다. val로 선언한 프로퍼티는 읽기전용, var로 선언한 프로퍼티는 쓰기전용이다. 따라서 val는 게터만 갖고, var는 둘다 갖는다.

코틀린의 프로퍼티도 기본으로 getter가 있지만, 이걸 커스텀 할 수도 있는데 이 프로퍼티를 backing field라고 하며 프로퍼티 값을 그때 그때 계산해서 사용하는 작업이 필요할때 사용할 수 있다. 

이런 값만 모아둔 클래스를 value object라고 하는데, 자바에서 getter와 setter로 점철된 코드가 있다면 코틀린에서는 data class라는 강력한 도구가 있다.

```kotlin
val isSquare: Boolean
    get(){
        return h == w
    }
```

get()을 붙여 선언하면 커스텀 접근자를 만들 수 있게된다. 이 프로퍼티는 h와 w가 같으면 true를 반환 하는 프로퍼티다.

### enum과 when

enum은 열거형 타입이고, when은 분기 문법이다.

```kotlin
enum class Color{
    RED, ORANGE, BLUE
}
```

enum class 라는 키워드로 enum객체를 만든다. 

enum class 안에 프로퍼티와 메서드를 선언 할 수 있는데, 이때 코틀린에서 유일하게 세미콜론이 사용된다. enum 상수를 정의할 때, 상수 목록 가장 끝에 세미콜론으로 구별 지어줘야 메서드 정의 부분을 인식한다.

이렇게 만든 enum 객체는 when에서 유용하게 쓰인다.

when 역시 반환값을 도출할 수 있는 식이고, switch와 비슷하지만 더욱 유연하게 작동한다. 

```kotlin
when(input){
    a -> "A"
    b,c -> "B"
    1...5 -> "NUM"
    else -> "O" 
}
```

when은 break를 따로 적어줄 필요가 없고, 비교 값의 타입이 달라도 되고, 범위로 비교할수도 있고, 여러개 값과 매치해 볼 수도 있다. 아주 유연하다.

`setOf()`를 사용해 집합으로 비교도 가능하다. 이 말은 객체 비교가 된다는 말이다. when은 인자가 있을 수도 있지만, 함수 반환 값으로 사용될 때는 함수의 파라미터를 바로 조건분기로 끌고과 인자 없이도 사용가능하다.

```kotlin
fun foo1(c1: Color, c2: Color) = when((c1 == "R")&&(c2 == "B") -> "P")
```

이런식으로 사용할 수 있다. 즉 각 분기의 조건이 boolean판별이면 가능한 방식이다.

분기에 블록을 사용하면 맨 마지막 줄에 return을 붙이지 않아도 반환값으로 인식한다.

### 타입 캐스팅

스마트 캐스트는 컴파일러가 프로그래머 대신 캐스팅을 해주는 것으로, is 검사를 통해 타입을 캐스팅하는 것이다. 검사를 마친 변수의 타입은 검사이후 계속 같은 타입을 갖는것으로 컴파일러가 인식한다. 따라서 val로 선언해줘야한다.

명시적인 타입 캐스팅은 `as 타입` 으로 해주면 된다.

### iteration

반복문의 사용법을 알려주는 절이다

`for(i in a..b)`와 같은 범위형태로 사용할 수 있고, 배열을 대상으로 foreach같이 돌릴 수도 있다.

범위 지정시 ..는 항상 [ ] 이다. `a until b` 를 사용하면 [ )로 끝 값을 포함하지 않게 된다. 

인덱스를 사용하려면 컬렉션에 .withIndext()와 인덱스 변수를 같이 넣어주면 되는데

```kotlin
for((idx, i) in list.withIndex()
```

이때 idx가 인덱스 번호를 따라간다.

반복문에서 사용되는 in은 연산자이다. 어떤 값이 해당 범위에 속하는지 in을 통해 true/false를 알 수 있다. when에서도 사용되는 것을 본 적이 있을 것이다.

### 예외처리

예외처리에 사용되는 try, throw 문 역시 코틀린에서는 식이다. 

```kotlin
val exc = try{
    ..
}catch(e: NumberFormatException){
    // 예외 발생
}finally{
    // 마무
}
```

예외처리할 때는 try~catch~finally를 사용할 수 있고, 함수가 반환값으로 예외를 포함할 의무가 없다. 자바로 사용하다보면 throws IOException을 사용할 때가 있는데 코틀린에는 그 작업이 필요없다는 뜻이다.
